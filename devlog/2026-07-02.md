## 补充更新 (18:00-19:30) — 分轨引擎升级

### 背景
Phase 1 的 htdemucs_6s 分轨质量不理想（尤其是吉他/钢琴 SDR 仅 3dB）。
调研了三个方案后，选择集成 `audio-separator` (PyPI 封装 UVR 70+ 模型)。

### 方案对比
| 方案 | 结论 |
|------|------|
| StemRoller | 仅 Demucs GUI 壳，不提升质量 |
| UVR GUI | 功能强但嵌入困难 |
| **audio-separator** | Python API + 70+ 模型 + ONNX/Roformer ✅ |

### 完成
- [x] `pip install audio-separator` (v0.44.2)
- [x] FFmpeg 加入 PATH (winget 安装但没配置 PATH)
- [x] 编写 `src/core/separation/audio_separator_runner.py`
  - 双策略: `vocal_priority` (默认) + `full_band`
  - 与旧 `demucs_runner.py` 接口兼容
- [x] 更新 `main.py` CLI: `-s/--strategy` 替代 `-m/--model`
- [x] 更新 `pipeline.py` 和 `settings.py`
- [x] 测试通过:
  - `full_band`: htdemucs_6s 6轨分离 (等价旧版)

## 补充更新 (22:00-23:45) — Phase 2: MIDI 转录 + 鼓组切片

### 策略
basic-pitch ONNX 模型已在 Spotify 官方包中 (`basic_pitch/saved_models/icassp_2022/nmp.onnx`)。
通过 `pip install basic-pitch --no-deps` 安装包体（不装 TensorFlow），直接使用 ONNX Runtime 推理。

### 完成
- [x] `pip install basic-pitch --no-deps` + `mir_eval` (ONNX 推理依赖满足)
- [x] 编写 `src/core/transcription/basic_pitch_onnx.py` — ONNX 推理封装
  - 包装 basic_pitch.inference.predict，支持 progress_callback
  - 输入: 音频文件 → 输出: MIDI + note_events + model_output
  - 参数: onset_threshold, frame_threshold, minimum_note_length 等可调
- [x] 编写 `src/core/transcription/drum_slicer.py` — 鼓组采样切片
  - librosa onset 检测 → 切片 → 分类 (kick/snare/hihat/tom/cymbal)
  - 频谱质心 + 频带能量比分类
  - 支持 export_drum_kit 导出精选鼓组
  - 测试: 15s 鼓组 → 28 个切片 (kick:11, cymbal:9)
- [x] 编写 `src/core/transcription/midi_cleaner.py` — MIDI 后处理
  - 量化 (snap to grid)、去噪 (移除碎音/弱音/孤立音)
  - 合并重叠同音音符、力度归一化
  - 调性估算 (Krumhansl-Kessler 算法)
- [x] 更新 `pipeline.py`: run_transcription + run_drum_slicing 实现
- [x] 更新 `main.py`: transcribe / drums CLI 子命令

### 测试验证
- `python main.py transcribe <stem>`: 钢琴 15s → 88 音符 MIDI ✅
- `python main.py drums <stem>`: 鼓组 15s → 28 切片, 2 类鼓组 ✅
- MIDI 输出验证: 88 音符, 音域 G2-D5, 时长 61-1207ms ✅
